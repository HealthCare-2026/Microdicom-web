Orthanc Architecture & Platforms

Orthanc เป็น DICOM Server แบบ Open-source ที่ออกแบบมาให้มีขนาดเล็ก (Lightweight) และช่วยลดความซับซ้อนของมาตรฐาน DICOM ทำให้สามารถเชื่อมต่อกับระบบภาพทางการแพทย์ และระบบสารสนเทศของโรงพยาบาลได้ง่ายขึ้น

Core Engine

โปรแกรมหลักของ Orthanc ถูกพัฒนาด้วยภาษา C++ เพื่อให้มีประสิทธิภาพสูงและใช้ทรัพยากรเครื่องน้อย โดยใช้ DCMTK Toolkit เป็นเครื่องมือหลักในการจัดการโปรโตคอล DICOM และการสื่อสารระหว่างระบบภาพทางการแพทย์

Performance Highlight

Lightweight design enables deployment even on small servers or embedded systems.

RESTful API Layer

Orthanc มี REST API ผ่านโปรโตคอล HTTP ซึ่งช่วยซ่อนความซับซ้อนของมาตรฐาน DICOM ทำให้นักพัฒนาสามารถเข้าถึงข้อมูลภาพทางการแพทย์ในรูปแบบ JSON, PNG หรือ DICOM

สามารถนำไปพัฒนาแอปพลิเคชันเพิ่มเติมด้วยภาษาโปรแกรมต่าง ๆ เช่น Python, JavaScript, PHP หรือ Java

Storage & Database

โดยค่าเริ่มต้น Orthanc ใช้ SQLite สำหรับจัดเก็บดัชนีของข้อมูล และเก็บไฟล์ภาพ DICOM ลงใน Local Filesystem โดยตรง จึงไม่จำเป็นต้องติดตั้ง Database Server เพิ่มเติม

นอกจากนี้ยังสามารถขยายระบบได้ผ่าน Plugin เพื่อเชื่อมต่อกับฐานข้อมูล เช่น PostgreSQL, MySQL หรือ MariaDB รวมถึงระบบ Cloud Storage

Supported Platforms

Orthanc ถูกออกแบบให้เป็นระบบ Cross-platform สามารถติดตั้งและใช้งานได้ในหลายระบบปฏิบัติการ

  • • Windows (มี Binary สำหรับ 32-bit และ 64-bit)
  • • Linux (รองรับหลาย Distribution เช่น Debian, Ubuntu, Fedora และ Arch Linux)
  • • macOS (สามารถรันบน OS X / macOS ได้)
  • • Docker Container ซึ่งเป็นวิธีที่นิยมสำหรับการ Deploy ระบบในปัจจุบัน
  • • ระบบตระกูล BSD เช่น FreeBSD และ OpenBSD
