Orthanc Architecture

Orthanc เป็น DICOM Server แบบ Open-source ที่ออกแบบมาให้มีขนาดเล็ก (Lightweight) และช่วยลดความซับซ้อนของมาตรฐาน DICOM ทำให้สามารถเชื่อมต่อกับระบบภาพทางการแพทย์
และระบบสารสนเทศของโรงพยาบาลได้ง่ายขึ้น

Core Engine

Orthanc Core Server พัฒนาด้วยภาษา C++ เพื่อให้มีประสิทธิภาพสูง และสามารถทำงานเป็น DICOM Server ขนาดเล็ก (lightweight) ได้

Orthanc รองรับการทำงานตามมาตรฐาน DICOM network operations เช่น

  • DICOM C-STORE (รับ/ส่งภาพ)
  • DICOM C-FIND (ค้นหาข้อมูล)
  • DICOM C-MOVE / C-GET (retrieve images)
  • DICOM C-ECHO (ตรวจสอบการเชื่อมต่อ)

Architecture Concept

Orthanc acts as a bridge between DICOM systems and modern web technologies.

RESTful API Layer

Orthanc มี REST API ที่ช่วยแปลงการทำงานของ DICOM ให้กลายเป็น Web Service ที่ใช้งานง่ายผ่าน HTTP

นักพัฒนาสามารถเข้าถึงข้อมูลภาพในรูปแบบ JSON, PNG, JPEG หรือ DICOM

  • Retrieve DICOM instances
  • Query study / patient metadata
  • Upload images through HTTP

REST API นี้สามารถใช้งานได้จากทุกภาษาโปรแกรมที่รองรับ HTTP และ JSON

Storage & Database

Orthanc ใช้ SQLite ทำหน้าที่เป็น default database/index backend สำหรับจัดเก็บดัชนีและ metadata ของ DICOM resources ขณะที่ไฟล์ DICOM จริงถูกเก็บใน filesystem

ระบบสามารถขยายได้ผ่าน plugin เพื่อใช้ฐานข้อมูลระดับ enterprise เช่น

  • PostgreSQL (official plugin)
  • MySQL (official plugin)
  • Cloud storage backend

โครงสร้างนี้ช่วยให้ Orthanc สามารถ scale ได้ตั้งแต่ small imaging lab จนไปถึง hospital PACS

SQLite เหมาะสำหรับระบบขนาดเล็ก (ประมาณ ≤ 50,000 DICOM instances)

Plugin Architecture

Orthanc มี Plugin System ที่ช่วยให้สามารถเพิ่มความสามารถใหม่ โดยไม่ต้องแก้ไข core server

Plugin ที่สำคัญใน ecosystem เช่น

  • DICOMweb plugin
  • Stone Web Viewer (official plugin)
  • Osimis Web Viewer
  • PostgreSQL database plugin
  • Artificial intelligence plugins (e.g., TCIA plugin)

โครงสร้างนี้ทำให้ Orthanc สามารถปรับใช้ได้ใน research, hospital PACS, AI imaging pipelines

Orthanc System Architecture

Orthanc เป็น DICOM Server ที่ถูกพัฒนาด้วยภาษา C++ และออกแบบให้เป็นระบบที่มีขนาดเล็ก (lightweight) แต่สามารถขยายความสามารถได้ผ่าน plugin ecosystem ทำให้สามารถนำไปใช้งานได้ทั้งในระบบ PACS ของโรงพยาบาล งานวิจัยทางการแพทย์ และระบบ Medical Imaging

  • Orthanc Core Server ทำหน้าที่เป็นศูนย์กลางในการจัดการ workflow ของข้อมูลภาพทางการแพทย์ และการสื่อสารระหว่างระบบ DICOM
  • DICOM Networking Support รองรับ DICOM network operations เช่น
    C-STORE, C-FIND, C-MOVE, C-Echo และ C-GET
  • HTTP Server ให้บริการ REST API เพื่อให้ Web applications สามารถเข้าถึงข้อมูลภาพได้ผ่าน HTTP
  • SQLite ทำหน้าที่เป็น default database/index backend สำหรับจัดเก็บดัชนีและ metadata ของ DICOM resources ขณะที่ไฟล์ DICOM จริงถูกเก็บใน filesystem
  • Orthanc Explorer เป็น built-in Web interface สำหรับดูภาพทางการแพทย์ผ่าน browser โดยไม่ต้องติดตั้ง software เพิ่ม

Supported Platforms

Orthanc รองรับการทำงานบนหลายระบบปฏิบัติการ โดยเน้นการใช้งานบน Windows และ Linux เป็นหลัก ในขณะที่ macOS มักใช้สำหรับ development และ testing นอกจากนี้ Orthanc ยังรองรับ DICOM networking workflow เช่น C-Echo, C-Store, C-Find และ C-Move

Windows

รองรับ Windows อย่างเป็นทางการ พร้อม binary package สำหรับใช้งานจริง เหมาะสำหรับ deployment ใน environment ทั่วไป

Linux

เป็น platform หลักสำหรับ production รองรับหลาย distribution เช่น Ubuntu, Debian และ Fedora เหมาะสำหรับ server และ cloud deployment

macOS

รองรับในลักษณะการ build จาก source code โดยใช้สำหรับ development และ testing เป็นหลัก ไม่ใช่ production platform หลัก

Docker

มี official Docker images สำหรับการ deploy แบบ containerized รองรับการใช้งานบน cloud และ orchestration เช่น Kubernetes
