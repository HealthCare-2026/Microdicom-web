User Interfaces Supported

Interfaces ที่ Orthanc รองรับ

Orthanc เป็น DICOM server และ medical image archive ที่รองรับการเข้าถึงข้อมูลภาพทางการแพทย์ได้หลายรูปแบบ โดยแบ่งออกเป็น 3 Interfaces หลักดังนี้ [1]

1. Web Interface

Orthanc provides a built-in web interface (accessible at port 8042 by default) that allows users to browse patients, studies, series, and instances stored in the DICOM server. Users can search patient records and view DICOM metadata directly through a browser without any additional software installation. [2]

  • เรียกดูข้อมูลผู้ป่วยและรายการตรวจผ่านเบราว์เซอร์
  • ค้นหา patients, studies, series และ instances ได้
  • ดู DICOM metadata ได้จากหน้าเว็บโดยตรง
  • เข้าถึงได้ที่ http://localhost:8042 (default port)

2. REST API

Orthanc offers a comprehensive RESTful API that returns data in JSON format, allowing external applications to access and manage DICOM data programmatically. The API runs on the same port as the web interface (default: 8042) and supports full CRUD operations on all DICOM resources. [3]

  • เหมาะสำหรับการเชื่อมต่อกับระบบภายนอก
  • เข้าถึงและจัดการข้อมูลแบบโปรแกรมได้ ผ่าน JSON format
  • รองรับงาน integration และ automation
  • ตัวอย่าง endpoint: GET /patients, GET /studies/{id}

3. DICOM Network Interface

Orthanc supports the standard DICOM network protocol (DIMSE) for communication with imaging devices such as CT, MRI, and PACS systems. It listens on port 4242 by default and supports core DICOM services including C-STORE, C-FIND, C-MOVE, and C-ECHO. [4]

  • สื่อสารกับอุปกรณ์ถ่ายภาพทางการแพทย์ตามมาตรฐาน DICOM
  • เชื่อมต่อกับ CT, MRI และ PACS ได้
  • รองรับ C-STORE, C-FIND, C-MOVE, C-ECHO
  • Default DICOM port: 4242
Guided Explanation

รายละเอียดของ Interfaces ที่รองรับ

Web Interface

เข้าถึงข้อมูล DICOM ผ่านเว็บเบราว์เซอร์ได้โดยตรง

ผู้ใช้สามารถเปิดเว็บ Interface ของ Orthanc เพื่อเรียกดูผู้ป่วย การตรวจ ชุดภาพ และรายละเอียดของไฟล์ DICOM ได้สะดวก เหมาะสำหรับการตรวจสอบข้อมูลเบื้องต้นและการดู metadata ในระบบ โดย Orthanc จะเปิด web interface ที่ http://localhost:8042 เป็นค่าเริ่มต้น [2]

ใช้งานผ่าน browser ไม่ต้องติดตั้งซอฟต์แวร์เพิ่ม ดูข้อมูลผู้ป่วยและการตรวจ ตรวจสอบ metadata ได้ทันที Default port: 8042
REST API

เชื่อมต่อข้อมูลกับระบบอื่นได้แบบโปรแกรม

REST API ของ Orthanc เหมาะสำหรับการพัฒนาแอปพลิเคชัน การทำ automation และการเชื่อมโยงระบบทางการแพทย์ ที่ต้องการเข้าถึงข้อมูล DICOM อย่างเป็นระบบ โดย API จะส่งข้อมูลในรูปแบบ JSON และรองรับ HTTP methods ครบถ้วน (GET, POST, PUT, DELETE) [3]

ตัวอย่าง endpoints: GET /patients — ดึงรายชื่อผู้ป่วยทั้งหมด | GET /studies/{id}/archive — ดาวน์โหลด study เป็น ZIP | POST /instances — อัปโหลด DICOM instance

รองรับ integration กับระบบภายนอก ข้อมูลส่งกลับเป็น JSON format เหมาะกับงาน backend และ tools รองรับ GET, POST, PUT, DELETE
DICOM Network Interface

สื่อสารกับอุปกรณ์และระบบภาพทางการแพทย์ตามมาตรฐาน

Orthanc รองรับการสื่อสารผ่าน DICOM network protocol (DIMSE) ซึ่งเป็นมาตรฐานสำคัญในงาน medical imaging ทำให้สามารถรับส่งข้อมูลกับเครื่องมือแพทย์และ PACS ได้ โดยรองรับ DICOM services หลัก ได้แก่ C-STORE (รับภาพ), C-FIND (ค้นหา), C-MOVE (ส่งภาพ) และ C-ECHO (ทดสอบการเชื่อมต่อ) [4]

ใช้มาตรฐาน DICOM (DIMSE) เชื่อมต่อ CT, MRI, PACS รองรับ C-STORE, C-FIND, C-MOVE, C-ECHO Default DICOM port: 4242
