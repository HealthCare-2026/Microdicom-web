Orthanc Server

Core Features

ฟีเจอร์สำคัญของ Orthanc DICOM Server สำหรับการจัดเก็บ จัดการ และเชื่อมต่อข้อมูลภาพทางการแพทย์

Reference: https://www.orthanc-server.com/

DICOM Image Storage

Feature: จัดเก็บและจัดการภาพทางการแพทย์ในรูปแบบ DICOM

Description: Orthanc สามารถทำหน้าที่เป็น DICOM Store หรือ Mini-PACS สำหรับเก็บภาพจากเครื่องมือแพทย์ เช่น CT, MRI และ X-Ray โดยไม่ต้องใช้ระบบ PACS ขนาดใหญ่

Example: เครื่อง CT Scanner ส่งภาพสแกนของผู้ป่วยมายัง Orthanc เพื่อจัดเก็บและเรียกดูในภายหลัง

RESTful API Integration

Feature: ควบคุมและดึงข้อมูลผ่าน REST API

Description: Orthanc มี RESTful API ในตัว ช่วยให้นักพัฒนาสามารถเชื่อมต่อระบบอื่น เช่น AI, EMR หรือ Web Application เพื่อดึงข้อมูล DICOM และ metadata ในรูปแบบ JSON

Example: ระบบ AI เรียก API ของ Orthanc เพื่อดึงภาพ X-Ray ไปวิเคราะห์โรคปอดอัตโนมัติ

Built-in Web Interface

Feature: เข้าถึงและจัดการข้อมูลผ่าน Web Interface

Description: Orthanc มีหน้าเว็บภายในระบบที่ช่วยให้ผู้ใช้สามารถค้นหา ดู และจัดการไฟล์ DICOM ผ่าน browser ได้โดยไม่ต้องติดตั้งโปรแกรมเพิ่มเติม

Example: แพทย์สามารถเปิด browser แล้วค้นหาภาพ MRI ของผู้ป่วยจาก Orthanc ได้ทันที

Plugin Extension System

Feature: ขยายความสามารถของระบบด้วย Plugin

Description: Orthanc รองรับ plugin ที่ช่วยเพิ่มฟีเจอร์ เช่น Web viewer, DICOMweb support หรือ database backend ทำให้สามารถปรับแต่งระบบให้เหมาะกับการใช้งานได้

Example: โรงพยาบาลติดตั้ง Web Viewer plugin เพื่อให้แพทย์ดูภาพ DICOM ผ่านเว็บได้

DICOM Network Communication

Feature: รองรับการรับ-ส่งข้อมูลผ่าน DICOM Protocol

Description: Orthanc รองรับคำสั่งสำคัญของ DICOM protocol เช่น C-Store, C-Find, C-Move และ C-Echo ทำให้สามารถเชื่อมต่อกับอุปกรณ์ทางการแพทย์ได้ตามมาตรฐาน

Example: เครื่อง X-Ray ส่งภาพผู้ป่วยไปยัง Orthanc โดยอัตโนมัติผ่าน C-Store

DICOM to Image Conversion

Feature: แปลงภาพ DICOM เป็น PNG หรือ JPEG

Description: Orthanc สามารถสร้างภาพ PNG หรือ JPEG จากไฟล์ DICOM ได้แบบ real-time ทำให้สามารถนำภาพไปแสดงในเว็บหรือแอปพลิเคชันทั่วไปได้ง่าย

Example: ระบบเว็บโรงพยาบาลแสดงภาพ X-Ray ของผู้ป่วยเป็น PNG บนหน้าประวัติผู้ป่วย

DICOMweb Support

Feature: รองรับมาตรฐาน DICOMweb

Description: Orthanc รองรับ QIDO-RS, WADO-RS และ STOW-RS ผ่าน plugin ทำให้ระบบสามารถค้นหา อัปโหลด และดาวน์โหลดภาพ DICOM ผ่าน HTTP/HTTPS ได้

Example: ระบบ Cloud PACS เชื่อมต่อกับ Orthanc ผ่าน WADO-RS เพื่อดึงภาพ MRI

Secure Data Transfer

Feature: รองรับการเข้ารหัสข้อมูลด้วย TLS

Description: Orthanc รองรับ TLS encryption สำหรับการรับ-ส่งข้อมูล DICOM เพื่อเพิ่มความปลอดภัยของข้อมูลผู้ป่วยในเครือข่ายโรงพยาบาล

Example: ภาพ MRI ถูกส่งจากเครื่องสแกนไปยัง Orthanc ผ่านการเชื่อมต่อที่เข้ารหัสด้วย TLS

Flexible Database Backend

Feature: รองรับฐานข้อมูลหลายประเภท

Description: Orthanc ใช้ SQLite เป็นค่าเริ่มต้น และสามารถเปลี่ยนไปใช้ PostgreSQL หรือ MySQL ผ่าน plugin เพื่อรองรับระบบที่มีข้อมูลขนาดใหญ่ขึ้น

Example: โรงพยาบาลขนาดใหญ่ใช้ PostgreSQL เพื่อจัดการข้อมูลภาพหลายล้านไฟล์

Orthanc Peer Synchronization

Feature: เชื่อมต่อหลาย Orthanc Server เข้าด้วยกัน

Description: Orthanc สามารถเชื่อมต่อกับ Orthanc server อื่น ๆ เพื่อส่งและซิงค์ข้อมูลภาพทางการแพทย์ระหว่างระบบได้โดยตรง

Example: โรงพยาบาลสาขาส่งภาพ CT Scan ไปยัง Orthanc ที่ศูนย์กลางของเครือโรงพยาบาล

Cross-Platform Deployment

Feature: ใช้งานได้บนหลายระบบปฏิบัติการ

Description: Orthanc สามารถติดตั้งได้บน Windows, Linux และ macOS รวมถึงรองรับ Docker เพื่อการ deploy ที่ง่ายและรวดเร็ว

Example: ทีม IT ติดตั้ง Orthanc ผ่าน Docker บน server Linux ของโรงพยาบาล

Open Source Platform

Feature: ซอฟต์แวร์ Open Source สำหรับงานภาพทางการแพทย์

Description: Orthanc เป็นซอฟต์แวร์ open source ที่สามารถนำไปใช้งาน ปรับแต่ง และพัฒนาต่อได้โดยไม่มีค่า license เหมาะทั้งสำหรับงานวิจัยและงานจริง

Example: มหาวิทยาลัยนำ Orthanc ไปใช้เป็นระบบจัดเก็บภาพสำหรับงานวิจัยด้าน AI ทางการแพทย์
