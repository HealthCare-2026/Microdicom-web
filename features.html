Orthanc Server

Core Features

ฟีเจอร์หลักของ Orthanc DICOM Server ที่ใช้สำหรับเก็บ จัดการ และเชื่อมต่อข้อมูลภาพทางการแพทย์

Source: https://www.orthanc-server.com/

DICOM Image Storage

Feature: การจัดเก็บภาพทางการแพทย์แบบ DICOM

Description: Orthanc สามารถทำหน้าที่เป็น DICOM storage หรือ Mini-PACS สำหรับเก็บภาพจากเครื่องมือทางการแพทย์ เช่น CT, MRI หรือ X-Ray ช่วยให้สามารถจัดการและค้นหาภาพทางการแพทย์ได้ง่ายขึ้น

RESTful API Integration

Feature: การเชื่อมต่อผ่าน REST API

Description: Orthanc มี REST API ในตัว ทำให้นักพัฒนาสามารถเชื่อมต่อระบบอื่น เช่น AI, EMR หรือ Web Application เพื่อดึงข้อมูลภาพ DICOM และ metadata ไปใช้งานต่อได้ง่าย

Orthanc Explorer (built-in web interface)

Feature: เว็บสำหรับจัดการข้อมูลในระบบ

Description: Orthanc มี Orthanc Explorer ซึ่งเป็นเว็บอินเตอร์เฟซที่มาพร้อมกับระบบ ผู้ใช้สามารถค้นหา ดู และจัดการไฟล์ DICOM ผ่านเว็บเบราว์เซอร์ได้ทันที โดยไม่ต้องติดตั้งโปรแกรมเพิ่มเติม

Plugin Extension System

Feature: เพิ่มความสามารถของระบบด้วย Plugin

Description: Orthanc รองรับ plugin เพื่อเพิ่มฟีเจอร์ใหม่ ๆ เช่น web viewer, DICOMweb หรือการเชื่อมต่อฐานข้อมูลอื่น ๆ ทำให้สามารถปรับระบบให้เหมาะกับการใช้งานของแต่ละองค์กรได้

DICOM Network Communication

Feature: การสื่อสารผ่าน DICOM Protocol

Description: Orthanc รองรับคำสั่งหลักของ DICOM ได้แก่ C-STORE, C-FIND, C-MOVE, C-GET และ C-ECHO ทำให้สามารถรับ-ส่งข้อมูลกับเครื่องมือแพทย์และระบบ PACS อื่น ๆ ได้ตามมาตรฐาน

DICOM to Image Conversion

Feature: แปลง DICOM เป็น PNG หรือ JPEG

Description: Orthanc สามารถสร้างภาพ PNG หรือ JPEG จากไฟล์ DICOM ได้แบบ on-the-fly ผ่าน REST API ทำให้สามารถนำภาพไปแสดงในเว็บหรือแอปพลิเคชันทั่วไปได้ง่าย

DICOMweb Support

Feature: รองรับมาตรฐาน DICOMweb

Description: Orthanc รองรับมาตรฐาน DICOMweb เช่น QIDO-RS, WADO-RS และ STOW-RS ผ่าน plugin ทำให้สามารถค้นหา อัปโหลด และดาวน์โหลดไฟล์ DICOM ผ่าน HTTP หรือ HTTPS ได้

Secure Data Transfer

Feature: การสื่อสารแบบปลอดภัย

Description: Orthanc รองรับ HTTPS/TLS encryption เพื่อเพิ่มความปลอดภัยในการรับ-ส่งข้อมูลระหว่างระบบ และช่วยปกป้องข้อมูลผู้ป่วยในเครือข่าย

Flexible Database Backend

Feature: รองรับฐานข้อมูลหลายแบบ

Description: Orthanc ใช้ SQLite เป็นค่าเริ่มต้น และสามารถเปลี่ยนไปใช้ PostgreSQL หรือ MySQL ผ่าน plugin เพื่อรองรับระบบที่มีข้อมูลจำนวนมาก

Orthanc Peers Communication

Feature: เชื่อมต่อ Orthanc หลายเครื่องเข้าด้วยกัน

Description: Orthanc สามารถตั้งค่า Orthanc server อื่น ๆ เป็น peers เพื่อให้สามารถส่งและแลกเปลี่ยนข้อมูลภาพทางการแพทย์ระหว่างเซิร์ฟเวอร์หลายเครื่องได้

Cross-Platform Deployment

Feature: ใช้งานได้หลายระบบปฏิบัติการ

Description: Orthanc สามารถติดตั้งได้ทั้งบน Windows, Linux และ macOS รวมถึงสามารถ deploy ผ่าน Docker ได้ ทำให้ติดตั้งและจัดการระบบได้สะดวก

Open Source Platform

Feature: ซอฟต์แวร์แบบ Open Source

Description: Orthanc ถูกเผยแพร่ภายใต้สัญญาอนุญาต GNU GPL v3 open-source license ผู้ใช้สามารถนำไปใช้งาน ปรับแต่ง และพัฒนาต่อยอดได้ตามต้องการ
